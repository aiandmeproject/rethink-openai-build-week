# CHOICE Case Study Seed

CHOICE is not part of the Rethink runtime and is not a selectable reasoning method.

It is a mature framework produced through the broader Rethink project and may serve as evidence that repeated routed-reasoning cycles can create a stable domain-specific artifact. Its value to this prototype is illustrative:

- an original problem can be reframed;
- assumptions can be challenged across cycles;
- decisions can be locked and reopened;
- a reusable framework can emerge from accumulated project learning.

This seed deliberately contains no runtime integration instructions. A future domain-specific plugin could invoke CHOICE explicitly, but ordinary Rethink projects must continue to route only through the core method registry.

See `docs/CHOICE_CASE_STUDY.md` for the case-study boundary.
