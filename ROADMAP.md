# Rethink Product Roadmap

This roadmap is ordered by product integrity, not dates. An item belongs in Core only when it strengthens the routed uncertainty-management loop; otherwise it should remain a clean module or product on top of Core.

## Rethink Core v0.1 — Build Week

Implemented and verified:

- PEC state, project lifecycle, and explicit disposition;
- STM highest-leverage unanswered question and trunk-before-branches routing;
- visible one-method-per-cycle router with human override;
- structured assumptions, evidence, evidence quality, links, locks, and traces;
- public-research and human/real-world evidence gates;
- persistent Lab Notebook and reason-gated human decisions;
- loop detection for no-new-evidence cycles;
- strict project-context isolation;
- deterministic no-key Demo Mode and server-side GPT-5.6 Live Mode;
- versioned reasoning-module registry;
- device-local persistence adapter, project import/export, and notebook export;
- container deployment path, health endpoint, security headers, and reviewer documentation.

Build Week exit work:

- execute a paid Live Mode acceptance call when a key is available;
- deploy the exact product to a controlled public URL;
- run the release checklist against that URL;
- record the final URL and demo-video evidence.

## Rethink Core v0.2 — Personal Daily Use

Candidates, subject to use and eval results:

- multi-project dashboard and repository operations: create, open, resume, archive, export, import;
- project summaries showing PEC phase, highest-leverage question, evidence state, last decision, next action, and last activity;
- stronger local or single-user durable persistence with tested migrations;
- automatic and scheduled backups with restore verification;
- resume summaries and project search;
- evidence deduplication, source freshness, threshold policies, and richer question linkage;
- tangible tangent promotion to a child project while preserving lineage;
- router-quality evals and regression fixtures from real projects.

Explicit cross-project transfer would remain opt-in: Rethink may suggest a relevant project, but information enters the active state only after user authorization and a recorded import event.

## Rethink Core v0.3 — Modular Reasoning Platform

Candidates after the Core contracts prove stable:

- signed/versioned module packages and compatibility checks;
- dynamic module installation outside a source-code release;
- Framework Builder for authoring and testing module contracts;
- explicit module dependencies and tool permissions;
- domain packs that register bounded modules without changing PEC, STM, Evidence, Notebook, or disposition reducers;
- authorized cross-project mechanism transfer with provenance;
- module-level evals, rollback, and version pinning.

This is not a marketplace commitment. The current registry is deliberately a small build-time extension seam.

## Future Products

### Rethink Research

Research that identifies the uncertainty and evidence threshold before searching, acquires available public evidence, preserves sources, updates assumptions, and distinguishes remaining real-world requirements.

### Rethink Validate

A focused `Validate this` product returning supported, partially supported, contradicted, inconclusive, public research required, real-world testing required, or currently unverifiable.

### Rethink Framework Builder

A lifecycle for turning expert reasoning into a purpose-trigger-input-process-output-failure-mode contract, testing overlap and safeguards, versioning it, and locking a compatible module.

### Rethink Cross-Domain Lab

A dedicated CDIL experience that transfers causal mechanisms—not surface vocabulary—with mandatory Transfer Validity checks.

### Rethink Organization Intelligence

A dedicated OIP experience for governance, incentives, observed behavior, mission/action alignment, partnership potential, competitive position, and gaps.

### Rethink Initiative Gate

A bounded gate that decides whether an idea deserves organized resources, with outcomes such as Project, Investigation, Tangent, Hold, Archive, or Kill.

CHOICE, Knowledge Confirmation, the AI-Native Education Framework, Metric Registry, and similar systems remain frameworks created **through** Rethink unless an explicit domain module invokes them. They are not universal Core operators.
